Glossary

 agent:agent-category  -  A concept scheme of categories of agent.
 agent:AgentCategory  -  The class of agent categories.
 community:Classification  -  A classification that describes members of a community.
 community:Community  -  The class of communities. A community is a group of persons and/or organizations that share common characteristics such as place, circumstance, etc.
 community:community-classification  -  A classification scheme for communities.
 dcterms:DCMIType  -  The set of classes specified by the DCMI Type Vocabulary, used to categorize the nature or genre of the resource.
 dcterms:DDC  -  The set of conceptual resources specified by the Dewey Decimal Classification.
 dcterms:IMT  -  The set of media types specified by the Internet Assigned Numbers Authority.
 dcterms:LCC  -  The set of conceptual resources specified by the Library of Congress Classification.
 dcterms:LCSH  -  The set of labeled concepts specified by the Library of Congress Subject Headings.
 dcterms:Location  -  dcterms:Location class fully represents the ISA Programme Location Core Vocabulary class of Location.
 dcterms:LocationPeriodOrJurisdiction  -  A location, period of time, or jurisdiction.
 dcterms:MESH  -  The set of labeled concepts specified by the Medical Subject Headings.
 dcterms:NLM  -  The set of conceptual resources specified by the National Library of Medicine Classification.
 dcterms:TGN  -  The set of places specified by the Getty Thesaurus of Geographic Names.
 dcterms:UDC  -  The set of conceptual resources specified by the Universal Decimal Classification.
 foaf:Agent  -  An agent (eg. person, group, software or physical artifact).
 foaf:Group  -  A class of Agents.
 foaf:Organization  -  An organization.
 foaf:Person  -  A person.
 foaf:Project  -  A project (a collective endeavour of some kind).
 geo:point  -  A point, typically described using a coordinate system relative to Earth, such as WGS84.
 geo:SpatialThing  -  Anything with spatial extent, i.e. size, shape, or position. e.g. people, places, bowling balls, as well as abstract areas like cubes.
 http://data.sedgemoor.gov.uk/def/project/proposal-status/accepted  -  The status of being accepted.
 http://data.sedgemoor.gov.uk/def/project/proposal-status/rejected  -  The status of being rejected.
 impact:ActualImpact  -  The class of actual impacts. An actual impact is a real impact that has occurred.
 impact:Impact  -  The class of impacts. An impact is a consequence. It may, for example, be a consequence of a large scale construction project, but there are many kinds of things that may have an impact. An impact has an impact on something. For example, an impact may have an impact on a community. However an impact can have an impact on a variety of different kinds of thing such as the environment or the economy.
 impact:impact-category  -  A concept scheme of impact categories.
 impact:ImpactCategory  -  The class of impact categories.
 impact:PredictedImpact  -  The class of predicated impacts. A predicted impact is a predication of an impact. A predicated impact is typically predicted to happen in the future.
 item:Category  -  The class of concepts in the Sedgemoor item taxonomy.
 item:Item  -  The class of items. This class is equivalent to rdfs:Resource. A type property on a resource can be used as a syntactic marker to support querying for resources with the marker.
 item:item-category  -  A SKOS concept scheme of concets for categorizing items.
 locn:Address  -  An "address representation" as defined in the INSPIRE Data Specification on Addresses, v3.0.1. The locn:addressId property may be used to link this locn:Address to other representations.
 locn:Geometry  -  The locn:Geometry class provides the means to identify a location as a point, line, polygon, etc. expressed using coordinates in some coordinate reference system.
 model:Prediction  -  The class of predications. A prediction is forcecast of some characteristic of the world. For example a weather forecast is a prediction. A prediction typically predicts some aspect of the world that will occur after the prediction is created.
 project:Project  -  The class of projects. This is a very general class intended to capture a very broad range of things that might be classed as projects from school homework assignments to major construction projects. Projects usually have one or more goals. Projects are of finite duration.
 project:Proposal  -  The class of proposals. A distinction is made between a proposal and a document or set of documents that describe the proposal. Instances of the class :Proposal correspond to the action of making or submitting a proposal. The wdrs:describedby property may be used to relate a proposal to one or more documents that describe the proposal that is made.
 project:proposal-status  -  A controlled vocabulary of proposal status values.
 project:ProposalStatus  -  The class of status's of a proposal.
 project:ResourceSource  -  The class of resource sources.
 skos:Collection
 skos:Concept
 skos:ConceptScheme
 skos:OrderedCollection
 version:Version  -  The class of versions. A version is a version of a versioned thing.
 version:VersionedThing  -  The class of versioned things. A versioned thing is a thing which may have more than one version.
 wdsr:Document  -  A POWDER document.
